## Scenario 1 — จัดการข้อมูลคู่ค้า (Vendor Master)
**🎯 วัตถุประสงค์:** แสดงการบริหารจัดการข้อมูลผู้ขายตั้งแต่รอตรวจอนุมัติจนถึงแบล็คลิสต์

**👤 Role:** Admin (nantaporn.s)  
**📍 เส้นทาง:** เมนู **คู่ค้า (Vendor) → ข้อมูลคู่ค้า**

| ขั้นตอน | Action | สิ่งที่ลูกค้าจะเห็น |
|---|---|---|
| 1 | เปิดหน้า ข้อมูลคู่ค้า | KPI cards: 7 ราย, 1 รอตรวจอนุมัติ, 3 ใช้งาน, 3 ระงับ/ปิดกั้น |
| 2 | คลิก Tab **"รอตรวจอนุมัติ (1)"** | กรองเหลือ บริษัท เซฟตี้เกียร์ ไทย จำกัด |
| 3 | คลิกปุ่ม **"ตรวจอนุมัติ"** (สีเหลือง) | เปิดหน้ารายละเอียดคู่ค้า |
| 4 | ดูข้อมูลทะเบียนพาณิชย์, เอกสาร, ที่อยู่ | เห็นข้อมูลครบถ้วน |
| 5 | กลับหน้ารายการ — คลิก Tab **"แบล็คลิสต์ (1)"** | เห็น บริษัท สมาร์ท ปริ้นติ้ง — สถานะแดง แบล็คลิสต์ |
| 6 | คลิก Tab **"ทั้งหมด"** — ชี้ dropdown กรองสถานะ | แสดงทุกสถานะในหน้าเดียว |

**💬 จุดเน้น:** "ระบบแยกสถานะผู้ขาย 6 ระดับ พร้อม SoD ป้องกันการอนุมัติตัวเอง"

| 3 | คลิกปุ่ม **"ส่งอนุมัติ"** | สถานะเปลี่ยนเป็น "รออนุมัติ" ทันที |
| 4 | คลิก Tab **"เกินงบประมาณ"** — หา PR2606011 (iPad Pro เกินงบแผนปี) | สถานะ "งบประมาณเกิน/ติดบล็อก" สีแดง |
| 5 | คลิกปุ่ม **"ขอยกเว้นงบ"** | alert "ส่งคำขอไปยัง CFO แล้ว" |
| 6 | คลิก Tab **"ไม่อนุมัติ"** — หา PR2606019 | คลิก **"ดูเหตุผล"** → alert เหตุผลการปฏิเสธ |
| 7 | คลิก Tab **"รออนุมัติ"** — คลิก **"อนุมัติด่วน"** | สถานะเปลี่ยนเป็น "อนุมัติแล้ว" ทันที |

**💬 จุดเน้น:** "ทุก PR มี Audit Trail — รู้ว่าใครอนุมัติ เมื่อไหร่ ด้วยเหตุผลอะไร"

## Scenario 4 — สร้างโครงการประกวดราคา (RFQ Bidding)
**🎯 วัตถุประสงค์:** แสดงการสร้าง RFQ หลายรูปแบบ — Sealed Bid, Live Auction, Evaluated

**👤 Role:** Admin / Buyer  
**📍 เส้นทาง:** เมนู **ประกวดราคา (Bidding)**

| ขั้นตอน | Action | สิ่งที่ลูกค้าจะเห็น |
|---|---|---|
| 1 | เปิดหน้า Bidding | KPI cards: 7 โครงการ, มีทุกสถานะ |
| 2 | ชี้ **RFQ2606005** (PendingEvaluation) | สถานะ "อยู่ระหว่างประเมิน" สีเหลือง |
| 3 | ชี้ **RFQ2606006** (NoAward) | สถานะ "ไม่มีผู้ชนะ" สีแดง + ปุ่ม "เปิดประมูลใหม่" |
| 4 | คลิก **"เปิดประมูลใหม่"** บน NoAward | alert ยืนยัน + สถานะกลับเป็น "เปิดรับซอง" |
| 5 | คลิกปุ่ม **"+ เปิดประมูลราคา (RFQ)"** | เปิดหน้า Create RFQ |
| 6 | ชี้ dropdown **ประเภทการประมูล (Bid Type)** | แสดงตัวเลือก: RFQ (แบบปิดทั่วไป), Sealed Bid, Open Auction, RFI, RFP |
| 7 | กลับหน้า List — คลิก **"ชวนผู้ขาย"** บนโครงการที่เปิดอยู่ | alert "ส่งคำเชิญให้ผู้ขายแล้ว" |

**💬 จุดเน้น:** "รองรับ 5 รูปแบบประมูล (RFQ, Sealed Bid, Open Auction, RFI, RFP) ตามนโยบายจัดซื้อ พร้อม Audit Trail ทุกขั้นตอน"

## Scenario 5 — เปรียบเทียบราคาและตัดสินผู้ชนะ (Bid Comparison)
**🎯 วัตถุประสงค์:** แสดง Committee Sealed-Bid Unlock + Multi-Criteria AI + การตัดสินผู้ชนะ

**👤 Role:** Admin  
**📍 เส้นทาง:** ประกวดราคา → **RFQ2606001** → คลิก "เปรียบเทียบและตัดสิน"

| ขั้นตอน | Action | สิ่งที่ลูกค้าจะเห็น |
|---|---|---|
| 1 | คลิก **"เปรียบเทียบ"** บน RFQ2606001 | หน้า Compare มี 3 Tab: **เปรียบเทียบซองปิด \| ประเมินคะแนนหลายมิติ \| จำลองการแข่งราคา** |
| 2 | อยู่ที่ Tab **"ซองปิด (Sealed Bid)"** — ดู Section คณะกรรมการ | เห็นรายชื่อกรรมการ 2 ท่าน + สถานะปลดรหัส |
| 3 | คลิกปุ่ม **"ปลดรหัสยืนยัน (ในฐานะกรรมการ)"** | กรอก passphrase → ซองเปิด → เห็นราคาผู้เสนอ |
| 4 | คลิกปุ่ม **"อนุมัติ Shortlist"** | สถานะ Shortlist เปลี่ยนเป็น อนุมัติแล้ว ✓ |
| 5 | คลิก Tab **"หลายเกณฑ์ (Multi-Criteria)"** | เห็นกระดาษให้คะแนน 4 หัวข้อ: เทคนิค 30 + ความรวดเร็วจัดส่ง 30 + การรับประกัน 20 + ความน่าเชื่อถือ 20 (เต็ม 100) — คำนวณรวมเป็นคะแนนเทคนิค |
| 6 | กลับ Tab ซองปิด → คลิกปุ่ม **"เลือกผู้ขายรายนี้"** บนผู้ชนะ | confirm dialog → สถานะ RFQ เปลี่ยนเป็น "ตัดสินแล้ว" |

**💬 จุดเน้น:** "ซองปิดต้องใช้กรรมการหลายคน unlock พร้อมกัน — ป้องกันการรู้ราคาล่วงหน้า 100%"

| 8 | ใส่เลขที่ Invoice ซ้ำ (เช่น INV-2026-0007) → Submit | **⛔ alert "ตรวจพบใบแจ้งหนี้ซ้ำ" — Duplicate Detection** |
| 9 | ใส่เลขใหม่ → Submit | Invoice บันทึกสำเร็จ |
